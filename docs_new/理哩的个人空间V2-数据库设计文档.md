Version: 2.0
Date: 2026-06-12

# 一、设计目标
V2 数据库围绕一个简单博客系统设计。

核心目标：

+ 统一文章模型
+ 支持分类、标签、归档
+ 支持 Markdown 正文
+ 支持封面图和媒体资源
+ 支持 Gallery 精选照片
+ 使用 Supabase Postgres 作为主数据存储
+ 使用 Supabase Storage 存储媒体文件
+ 支持自建账号密码登录
+ 支持管理员和普通用户角色
+ 支持公开 / 仅自己可见文章
+ 支持草稿、发布、归档状态

本版不使用：

+ Supabase Auth
+ 邮箱验证码登录
+ friends 好友分组权限模型
+ access_requests 权限申请
+ Travel / Essay / Journal / Moment 内容枚举

说明：

+ Supabase 仍然是数据库和文件存储服务。
+ 只是认证不使用 Supabase Auth，而是在业务数据库中维护用户、密码哈希和 session。

# 四、枚举设计
## post_status
```plain
draft
published
archived
```

说明：

+ `draft`：草稿
+ `published`：已发布
+ `archived`：归档隐藏

## post_visibility
```plain
public
private
```

说明：

+ `public`：公开文章。
+ `private`：仅自己可见文章，只有管理员登录后可见。

## user_role
```plain
admin
reader
```

说明：

+ `admin`：管理员，可以访问后台和前台全部文章。
+ `reader`：普通注册用户，当前只能阅读公开文章。

# 五、表结构
## 1. users
存储管理员和普通用户账号。

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | uuid | 主键 |
| username | text | 登录账号，唯一 |
| password_hash | text | 密码哈希 |
| display_name | text | 显示名 |
| role | user_role | 用户角色 |
| created_at | timestamptz | 创建时间 |
| updated_at | timestamptz | 更新时间 |

管理员初始数据：

```plain
username = root
password = 1212
role = admin
```

注意：

+ 数据库只保存 `1212` 的哈希值。
+ 不保存明文密码。
+ 普通用户注册时写入 `role = reader`。

## 2. sessions
存储账号登录会话。

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | uuid | 主键 |
| user_id | uuid | 关联 users.id |
| token_hash | text | session token 哈希 |
| expires_at | timestamptz | 过期时间 |
| created_at | timestamptz | 创建时间 |
| last_seen_at | timestamptz | 最近使用时间 |

说明：

+ 浏览器 cookie 只保存 session token。
+ 数据库保存 token 哈希。
+ 后端通过 session 找到当前用户和角色。

## 3. categories
存储文章主分类。

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | uuid | 主键 |
| name | text | 分类名称 |
| slug | text | URL 标识，唯一 |
| description | text | 分类说明 |
| sort_order | int | 排序 |
| visible | boolean | 是否在前台展示 |
| created_at | timestamptz | 创建时间 |
| updated_at | timestamptz | 更新时间 |

初始分类建议：

```plain
游游逛逛
卿卿我我
少侠
浮光片影
但是还有书籍
四季流转
花花草草
网站日记
码农小知识
未分类
```

## 4. tags
存储文章标签。

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | uuid | 主键 |
| name | text | 标签名称 |
| slug | text | URL 标识，唯一 |
| created_at | timestamptz | 创建时间 |

## 5. posts
存储文章正文和元信息。

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | uuid | 主键 |
| author_id | uuid | 关联 users.id |
| category_id | uuid | 关联 categories.id |
| title | text | 标题 |
| slug | text | URL 标识，唯一 |
| excerpt | text | 摘要 |
| markdown | text | Markdown 正文 |
| html | text | 渲染后的 HTML，可选缓存 |
| cover_asset_id | uuid | 封面图，关联 media_assets.id |
| cover_url | text | 外链封面地址，可选 |
| status | post_status | 文章状态 |
| visibility | post_visibility | 文章可见性 |
| noindex | boolean | 是否不在首页列表展示 |
| published_at | timestamptz | 发布时间 |
| created_at | timestamptz | 创建时间 |
| updated_at | timestamptz | 更新时间 |

说明：

+ `markdown` 是编辑器主存储字段。
+ `html` 可作为渲染缓存，不作为源数据。
+ `cover_asset_id` 用于站内上传图片。
+ `cover_url` 用于兼容外链图片。
+ `noindex` 参考 `lilisong` 主题逻辑，可用于不进入首页列表的特殊文章。
+ `visibility = private` 的文章只有管理员登录后能在前台看到。

## 6. post_tags
文章标签关系表。

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| post_id | uuid | 文章 ID |
| tag_id | uuid | 标签 ID |

联合主键：

```plain
(post_id, tag_id)
```

## 7. media_assets
存储图片、音频、视频等媒体资源元信息。

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | uuid | 主键 |
| uploader_id | uuid | 关联 users.id |
| post_id | uuid | 可选关联文章 |
| file_name | text | 原始文件名 |
| mime_type | text | MIME 类型 |
| storage_path | text | 存储路径 |
| public_url | text | 访问地址 |
| bucket | text | Supabase Storage bucket |
| alt | text | 图片替代文本 |
| caption | text | 图片说明 |
| featured | boolean | 是否进入 Gallery |
| width | int | 图片宽度 |
| height | int | 图片高度 |
| size_bytes | int | 文件大小 |
| created_at | timestamptz | 创建时间 |
| updated_at | timestamptz | 更新时间 |

Gallery 规则：

```plain
media_assets where featured = true
```

## Storage Bucket 设计
继续使用 Supabase Storage 存储媒体文件。

推荐 bucket：

```plain
public-media
private-media
```

说明：

+ `public-media`：公开文章图片、公开封面图、Gallery 图片。
+ `private-media`：仅自己可见文章中的图片和附件。
+ `media_assets.bucket` 记录文件所在 bucket。
+ `media_assets.storage_path` 记录 bucket 内路径。
+ `public-media` 可以公开读取。
+ `private-media` 不直接公开读取，由服务端在管理员登录后生成受控访问地址。
+ 如果一张图片被标记为 `featured = true` 进入 Gallery，应存放在 `public-media`。

## 8. site_settings
存储站点基础配置。

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| key | text | 主键 |
| value | jsonb | 配置值 |
| updated_at | timestamptz | 更新时间 |

示例配置：

```plain
site_title
site_description
home_intro
default_cover_url
navigation
```

# 六、SQL Schema 初版
```sql
create extension if not exists "pgcrypto";

create type post_status as enum (
  'draft',
  'published',
  'archived'
);

create type post_visibility as enum (
  'public',
  'private'
);

create type user_role as enum (
  'admin',
  'reader'
);

create or replace function set_updated_at()
returns trigger as $$
begin
  new.updated_at = now();
  return new;
end;
$$ language plpgsql;

create table users (
  id uuid primary key default gen_random_uuid(),
  username text not null unique,
  password_hash text not null,
  display_name text not null default '理哩',
  role user_role not null default 'reader',
  created_at timestamptz not null default now(),
  updated_at timestamptz not null default now()
);

create trigger users_set_updated_at
before update on users
for each row execute function set_updated_at();

create table sessions (
  id uuid primary key default gen_random_uuid(),
  user_id uuid not null references users(id) on delete cascade,
  token_hash text not null unique,
  expires_at timestamptz not null,
  created_at timestamptz not null default now(),
  last_seen_at timestamptz
);

create table categories (
  id uuid primary key default gen_random_uuid(),
  name text not null unique,
  slug text not null unique,
  description text,
  sort_order int not null default 0,
  visible boolean not null default true,
  created_at timestamptz not null default now(),
  updated_at timestamptz not null default now()
);

create trigger categories_set_updated_at
before update on categories
for each row execute function set_updated_at();

create table tags (
  id uuid primary key default gen_random_uuid(),
  name text not null unique,
  slug text not null unique,
  created_at timestamptz not null default now()
);

create table posts (
  id uuid primary key default gen_random_uuid(),
  author_id uuid not null references users(id) on delete restrict,
  category_id uuid references categories(id) on delete set null,
  title text not null,
  slug text not null unique,
  excerpt text,
  markdown text not null default '',
  html text,
  cover_asset_id uuid,
  cover_url text,
  status post_status not null default 'draft',
  visibility post_visibility not null default 'public',
  noindex boolean not null default false,
  published_at timestamptz,
  created_at timestamptz not null default now(),
  updated_at timestamptz not null default now(),
  constraint posts_publish_requires_published_at check (
    status <> 'published' or published_at is not null
  )
);

create trigger posts_set_updated_at
before update on posts
for each row execute function set_updated_at();

create table post_tags (
  post_id uuid not null references posts(id) on delete cascade,
  tag_id uuid not null references tags(id) on delete cascade,
  primary key (post_id, tag_id)
);

create table media_assets (
  id uuid primary key default gen_random_uuid(),
  uploader_id uuid not null references users(id) on delete restrict,
  post_id uuid references posts(id) on delete set null,
  file_name text not null,
  mime_type text not null,
  bucket text not null default 'public-media',
  storage_path text not null,
  public_url text not null,
  alt text,
  caption text,
  featured boolean not null default false,
  width int,
  height int,
  size_bytes int,
  created_at timestamptz not null default now(),
  updated_at timestamptz not null default now()
);

create trigger media_assets_set_updated_at
before update on media_assets
for each row execute function set_updated_at();

alter table posts
add constraint posts_cover_asset_fk
foreign key (cover_asset_id)
references media_assets(id)
on delete set null;

create table site_settings (
  key text primary key,
  value jsonb not null,
  updated_at timestamptz not null default now()
);
```

# 七、索引建议
```sql
create index posts_status_published_at_idx
on posts (status, published_at desc);

create index posts_public_feed_idx
on posts (status, visibility, noindex, published_at desc);
```

# 八、数据访问规则
前台：

+ 未登录访客：只能读取 `status = 'published' and visibility = 'public'` 的文章。
+ 普通登录用户：当前与访客一致，只能读取公开已发布文章。
+ 管理员登录用户：前台可以读取全部文章，包括 `private`、`draft`、`archived`。
+ 首页公开视图排除 `noindex = true` 的文章。
+ 管理员前台视图可以显示 `noindex = true` 的文章，但需要给出状态标识。
+ 分类、标签、归档在公开视图中只展示公开已发布文章；管理员视图展示全部并标识状态。

后台：

+ 需要管理员登录。
+ 可读写全部文章、分类、标签、媒体和设置。

注册登录：

+ 注册用户默认 `role = reader`。
+ `role = admin` 只允许通过数据库初始化或后台管理员操作创建。
+ 登录成功后创建 `sessions` 记录。
+ 退出登录时删除当前 `sessions` 记录。
