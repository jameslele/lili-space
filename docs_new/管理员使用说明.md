# 管理员使用说明

本文面向「理哩的个人空间」管理员，用于上线后日常维护文章、媒体、分类、标签、站点文案和用户。

## 登录

- 后台入口：`/admin`
- 未登录访问后台会跳转到 `/login`
- 普通用户访问后台会跳转到 `/forbidden`

初始化数据里保留了默认管理员约定：`root / 1212`。

生产上线前必须修改默认管理员密码，或创建新的强密码管理员后停用默认账号。

## 后台模块

- Dashboard：查看文章、分类、标签、媒体、用户等基础统计和最近文章。
- 文章：查看文章列表，按状态或可见性筛选，批量删除，进入编辑页。
- 分类：新增、编辑、隐藏分类。分类用于文章主栏目，一篇文章只有一个主分类。
- 标签：新增、编辑标签。标签用于文章关键词，一篇文章可以有多个标签。
- 媒体：上传和管理图片、音频、视频，编辑 alt / caption，删除媒体。
- Gallery：管理照片页精选图片，调整展示顺序。
- 站点设置：修改首页、页脚、关于页、文章版权说明等站点文案。
- 用户：查看用户，删除普通用户。管理员账号不能在此处被删除。

## 文章管理

新建文章：

1. 进入 `/admin/posts/new`。
2. 填写标题、摘要、正文、分类、标签、封面和可见性。
3. 先保存草稿，确认无误后再发布。

编辑文章：

1. 进入 `/admin/posts`。
2. 点击文章进入编辑页。
3. 修改内容后选择保存草稿、发布或归档。

正文编辑：

- 正文使用 Milkdown Crepe 编辑器。
- 数据库存储 source of truth 是 `posts.markdown`。
- 不在正文中写入颜色、字号、字体等样式配置。
- 前台样式由 `MarkdownContent` 统一控制。

媒体插入：

- 可以在文章编辑页上传媒体并插入正文。
- 图片会以 Markdown 图片插入。
- 音频和视频会以原生 HTML 标签插入，前台使用浏览器原生播放器。
- private-media 内容在管理员视图下通过 signed URL 展示。

封面设置：

- 可以手动填写封面 URL。
- 也可以从媒体里选择图片设为封面。
- 封面 URL 存在 `posts.cover_url`。

可见性和状态：

- `public`：公开内容，访客和普通用户在已发布状态下可见。
- `private`：仅管理员可见。
- `draft`：草稿，仅管理员可见。
- `published`：已发布。
- `archived`：归档，仅管理员可见。
- `show_on_home`：控制是否显示在首页。访客和普通用户不会看到隐藏首页的文章。

分类和标签：

- 分类是数据库数据，不是代码常量。
- 一篇文章只能选择一个主分类。
- 没有分类时使用「未分类」。
- 标签可以为空，也可以选择多个。
- 文章编辑页支持即时创建新分类和新标签。

## 媒体管理

Storage bucket：

- `public-media`：公开媒体，访客可以直接访问。
- `private-media`：私密媒体，必须保持非公开。

上传：

- 支持单个媒体上传。
- 支持批量上传图片、音频、视频。
- 图片大小限制为 10MB。
- 音频和视频大小限制为 50MB。
- 当前只做上传和原生播放，不做转码、抽帧或码率处理。

正文插入：

- 文章编辑页可以把媒体插入当前正文。
- signed URL 只用于管理员预览，不写入数据库。

Gallery：

- 只有图片可以加入 Gallery。
- 可以在媒体上传时标记精选。
- 可以在 Gallery 后台调整精选图片排序。
- 前台 `/gallery` 只给访客展示公开精选图片；管理员可以看到授权的私密图片。

## 前台管理员视图

管理员登录后访问前台，可以看到：

- 公开文章
- 私密文章
- 草稿文章
- 归档文章

状态 badge 含义：

- 公开：访客可见的已发布内容。
- 仅自己可见：只有管理员可见。
- 草稿：尚未发布。
- 已归档：不向访客展示的归档内容。
- 不显示在首页：不会进入访客首页列表。

## 普通用户

普通用户可以注册、登录、退出。

当前普通用户主要是为未来评论、点赞等功能预留身份体系。当前版本中，普通用户和访客一样，只能查看公开且已发布的内容，不能进入后台，也不能查看私密、草稿或归档文章。
