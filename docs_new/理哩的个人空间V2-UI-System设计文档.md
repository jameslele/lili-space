# 理哩的个人空间 V2 UI System 设计文档

Version: 2.0
Date: 2026-06-12

---

# 一、设计目标
V2 视觉系统延续 V1 的安静、文学、私人、杂志感，同时更贴近 `lilisong` 博客的真实内容结构。

页面应服务于阅读，而不是展示功能。

核心原则：

+ 文章优先
+ 图片克制使用
+ 分类和标签轻量呈现
+ 首页像博客首页，不像产品落地页
+ 后台安静实用，不做复杂 CMS 感

技术约束：

+ 前台使用 Astro 构建内容页面。
+ 局部交互使用 React Islands。
+ 数据来自 Supabase Postgres。
+ 媒体资源来自 Supabase Storage。
+ 登录态由自建 session 控制，不使用 Supabase Auth UI。
+ 后台编辑器使用 Milkdown Crepe。
+ 样式使用 Tailwind CSS。
+ 颜色、字体、阴影等全局 token 可用 CSS Variables 辅助定义，再映射到 Tailwind theme。



+ 后台状态标签可以使用状态色，但面积要小。

# 七、组件规范
## 1. Header
前台 Header：

+ 左侧站点名
+ 右侧导航
+ 移动端折叠菜单

导航建议：

```plain
首页
分类
归档
照片
关于
登录 / 用户菜单
```

不再放 Travel / Essays / Journal / Moments。

管理员登录后可额外显示：

```plain
后台
退出
```

## 2. 文章条目
文章条目是前台核心组件。

结构：

```plain
标题
日期 / 分类
摘要
标签
封面缩略图（可选）
```

风格：

+ 可有轻微分隔线
+ 不做厚重卡片
+ 封面图不喧宾夺主

## 3. Category Chip
用于展示分类。

视觉：

+ 小字号
+ 低饱和背景
+ 圆角 999px
+ 可点击

## 4. Tag
用于展示标签。

视觉：

+ 文本前可带 `#`
+ 比分类更弱
+ 不抢文章标题层级

## 5. Button
后台按钮分为：

+ Primary：发布、保存
+ Secondary：预览、取消
+ Danger：删除

前台尽量少用按钮，多用文字链接。

## 6. Status Badge
用于管理员前台视图和后台列表。

类型：

```plain
公开
仅自己可见
草稿
已发布
已归档
```

视觉：

+ 小尺寸
+ 弱背景
+ 不使用强烈警告色
+ 与标题保持距离，避免破坏阅读感

普通访客和普通用户前台不显示 `仅自己可见`、`草稿`、`已归档` 文章，因此也不会看到这些标识。

## 7. Form
后台表单需要稳定、清晰：

+ label 在上
+ 输入框高度 40px
+ 错误信息放在字段下方
+ 保存状态在页面顶部或按钮旁显示

# 八、后台 UI 原则
后台服务写作，不服务运营。

要求：

+ 左侧窄导航
+ 中间主工作区
+ 编辑页尽量减少干扰
+ Milkdown Crepe 编辑器区域要宽，元信息放侧栏或顶部折叠区
+ 文章可见性用清晰的二选一控件：公开 / 仅自己可见
+ 用户管理保持轻量，只展示账号、显示名、角色和创建时间

后台不要做：

+ 数据大屏
+ 复杂图表
+ 多层菜单
+ 过度装饰

# 九、响应式规则
## 桌面端
+ 内容居中
+ 顶部横向导航
+ 后台双栏布局

## 移动端
+ 前台导航折叠
+ 文章列表单列
+ 封面图在标题下方或隐藏为小图
+ 后台可用但不作为主要编辑场景

# 十一、登录与权限相关 UI
## 1. 登录页
登录页保持极简：

```plain
标题
账号输入框
密码输入框
登录按钮
注册链接
```

不使用第三方登录按钮。

## 2. 注册页
注册页字段：

```plain
用户名
显示名，可选
密码
确认密码
注册按钮
```

## 3. 管理员前台视图
管理员访问前台时，页面仍保持博客阅读体验，只增加必要标识：

+ 顶部显示后台入口。
+ 私密文章展示 `仅自己可见`。
+ 草稿展示 `草稿`。
+ 归档文章展示 `已归档`。

这些标识只为管理员判断内容状态，不作为普通读者体验的一部分。
