Version: 2.0
Date: 2026-06-12

# 一、后台目标
后台主要服务理哩本人。

目标是：

+ 快速登录
+ 快速写文章
+ 管理分类和标签
+ 上传和复用媒体
+ 控制 Gallery 展示
+ 调整站点基础信息
+ 查看普通注册用户

后台不是传统 CMS，不做复杂审批和多用户协作。

技术基础：

+ 前台框架：Astro
+ 数据存储：Supabase Postgres
+ 文件存储：Supabase Storage
+ 鉴权方式：自建账号密码登录，不使用 Supabase Auth
+ 编辑器：Milkdown Crepe
+ 正文格式：Markdown

# 三、登录设计
## 1. 登录方式
账号密码登录，不使用 Supabase Auth。

管理员初始账号：

```plain
root
```

初始密码：

```plain
1212
```

## 2. 登录页字段
```plain
账号
密码
登录按钮
错误提示
```

## 3. 登录成功
管理员登录成功跳转：

```plain
/admin
```

普通用户登录成功跳转：

```plain
/
```

## 4. 登录失败
提示：

```plain
账号或密码不正确
```

不提示具体是账号错误还是密码错误。

## 5. 会话
建议使用服务端 session 或 httpOnly cookie。

要求：

+ Cookie 使用 httpOnly。
+ 密码只在服务端校验。
+ 数据库保存密码哈希。
+ session 记录保存在 Supabase Postgres 的 `sessions` 表。

## 6. 普通用户注册登录
V2 需要提供普通用户注册和登录能力，为未来评论、点赞等功能预留。

普通用户注册字段：

```plain
用户名
密码
显示名，可选
```

普通用户默认角色：

```plain
reader
```

限制：

+ 普通用户不能访问 `/admin`。
+ 普通用户当前只能阅读公开文章。

# 七、文章编辑页
## 1. 页面布局
桌面端推荐：

```plain
顶部操作栏
左侧 / 主区域：编辑器
右侧：文章设置
```

## 2. 顶部操作栏
```plain
返回
保存草稿
预览
发布 / 更新
```

## 3. 主编辑区
使用 Milkdown Crepe。

正文源数据保存为 Markdown。

编辑器能力：

+ 标题
+ 段落
+ 加粗
+ 斜体
+ 链接
+ 引用
+ 列表
+ 图片
+ 分割线
+ 代码块

## 4. 文章设置
字段：

```plain
标题
Slug
摘要
分类
标签
封面图
发布时间
可见性
是否显示在首页
状态
```

说明：

+ 分类单选。
+ 标签多选，可新建。
+ 可见性支持 `公开` 和 `仅自己可见`。
+ `是否显示在首页` 对应 `noindex` 的反向含义。

## 5. 可见性逻辑
文章可见性字段：

```plain
public
private
```

后台文案：

```plain
公开
仅自己可见
```

规则：

+ `公开`：访客、普通登录用户、管理员均可在前台看到。
+ `仅自己可见`：只有管理员登录后可在前台看到。
+ 普通用户登录状态不影响文章可见性，本版不设计好友可见。

## 6. 管理员前台预览
管理员登录状态下访问前台博客页面时：

+ 可以看到全部文章。
+ 私密文章显示 `仅自己可见` 标识。
+ 草稿显示 `草稿` 标识。
+ 归档文章显示 `已归档` 标识。

未登录或普通用户登录状态下访问前台时：

+ 只能看到公开且已发布文章。
+ 直接访问私密文章 URL 时显示 404 或无权限页面。

# 八、Milkdown Crepe 选型说明
## 1. 选择原因
+ 原生面向 Markdown。
+ 比 Notion-like 块编辑器更贴近博客写作。
+ 能提供比普通 textarea 更舒适的编辑体验。
+ 正文可以直接以 Markdown 形式存储和迁移。
+ 后续如需导入 / 导出文章更自然。

## 2. V2 使用边界
V2 不追求完整复杂编辑器。

不做：

+ 多人协作
+ 评论批注
+ 复杂块数据库
+ 自定义 Notion Block
+ 富文本私有格式存储

## 3. 数据流
```plain
Milkdown Crepe 编辑
        ↓
Markdown 字符串
        ↓
posts.markdown
        ↓
前台渲染为 HTML
```

# 十一、Media 媒体管理
## 页面能力
```plain
上传
预览
复制地址
编辑 alt
编辑 caption
删除
标记为 Gallery
```

## 支持类型
V2 优先支持：

+ 图片

可兼容：

+ 音频
+ 视频

音频和视频先允许作为普通媒体插入文章，不单独设计播放器管理。

## 存储规则
媒体文件继续使用 Supabase Storage。

推荐 bucket：

```plain
public-media
private-media
```

规则：

+ 公开文章、公开封面、Gallery 图片进入 `public-media`。
+ 仅自己可见文章中的图片进入 `private-media`。
+ 后台插入图片时根据文章可见性选择默认 bucket。
+ 如果私密文章改为公开文章，需要提示是否迁移相关媒体到 `public-media`。

# 十五、验收标准
+ 未登录不能进入 `/admin`。
+ 可以用 `root / 1212` 完成登录。
+ 普通用户可以注册、登录、退出，但不能进入后台。
+ 可以创建 Markdown 文章并保存草稿。
+ 可以发布文章。
+ 可以设置分类、标签、封面、摘要和可见性。
+ 可以上传图片并插入正文。
+ 可以标记图片进入 Gallery。
+ 管理员登录状态下访问前台可以看到全部文章。
+ 非管理员状态下访问前台只能看到公开已发布文章。
+ 后台最终编辑器为 Milkdown Crepe。
